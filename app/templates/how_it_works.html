How It Works

From symptoms to prediction — the full pipeline explained.

The Prediction Pipeline

1

Input

You enter symptoms via text description, checkbox selection, or camera capture. Text input is processed with fuzzy NLP matching; camera frames are analyzed by a computer vision model.

2

Tokenization

Matched symptom keywords are joined into a text string and tokenized by DistilBERT's tokenizer — splitting words into subword tokens and mapping them to integer IDs.

3

Transformer Inference

Tokens are passed through the fine-tuned DistilBERT model. The [CLS] token's final embedding is fed into a classification head that outputs 42 logit scores — one per disease.

4

Softmax Ranking

Logits are converted to probabilities via softmax (summing to 100%). The top 3 diseases by probability are returned as the prediction result.

5

Result Enrichment

Each predicted disease is matched to its description and recommended precautions from our medical reference dataset, giving you actionable information alongside the diagnosis.

📚

Training Data

4,920 disease-symptom examples across 42 unique diseases. Each example combines up to 17 symptoms as a comma-separated text string.

🤖

Base Model

distilbert-base-uncased — a lightweight 66M parameter BERT variant pretrained on English Wikipedia and BooksCorpus, fine-tuned for medical classification.

🎯

Training Setup

3 epochs, batch size 32, learning rate 2e-5, max token length 64. Evaluated on a 20% held-out validation split after each epoch.
